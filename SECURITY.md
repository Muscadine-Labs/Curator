# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.

If you discover a vulnerability in Curator, report it privately by emailing:

muscadinelabs@gmail.com

Please include:
- Type of issue (e.g., XSS, injection, DoS)
- Affected components or file paths
- Steps to reproduce
- Proof-of-concept (if available)
- Potential impact assessment

## Response Timeline

- We aim to acknowledge reports within 5 business days.
- We will provide updates during remediation.
- We may follow up for clarification.

## Responsible Disclosure

We request that vulnerabilities not be publicly disclosed until a fix has been deployed.

## Safe Harbor

We support safe harbor for researchers who act in good faith, avoid privacy violations, data destruction, or service disruption, and follow this policy.
